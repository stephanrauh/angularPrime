Description Dropdown with skinning capabilities, editing, effects, filtering and custom content display.
Supports
  • Integrated with ng-disabled
  • Integrated with ng-options
  • Editable to input custom value
  • Dropdown items can be filtered
  • User callback function possible when selection is made
  • Custom content
Known issues
History
  • 0.6
    • Custom content can be defined with pui-content widget
Planned
  • Disabled option items
  • Source option of PrimeUI widget not supported
Configuration
Name Type Default Description
editable boolean false Allows entering a custom value in the 'dropdown' field
effect string fade Animation to run on displaying the options overlay, refer to jQuery UI effect api for valid values.
effectSpeed int/string normal Duration in milliseconds or keywords like slow, normal and fast.
filter boolean false Adds a filter input at options panel when enabled.
filterMatchMode string startsWith Mode of filter, valid values are startsWith, contains, endsWith and custom.
caseSensitiveFilter boolean false Defines case sensitivity on filtering.
filterFunction function null Function to be used in custom filtering, this function gets option as parameter and needs to return a boolean.
scrollHeight integer 200 Maximum height boundary of overlay panel in pixels, if total height of items exceed this value, scrollbar is displayed.
The custom content can be defined by using an element pui-content nested within the pui-dropdown element/tag.
The content can be any valid HTML where you can use the placeholders %LABEL% and %VALUE%.
